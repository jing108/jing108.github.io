  4. SerialExecutor是一个内部类：

    SerialExecutor的execute方法将传入的mFuture参数又包装了一层，将包装后的Runnable加入到自己的任务队列mTasks中。至于为什么要包装一层，下面再说。

    接着，判断mActive如果为null，则调用scheduleNext方法，该方法的作用是从队列头部取出一个任务，将其扔到线程池中运行，也就是说：mTasks任务队列中的Runnable都是运行在子线程中的。

    为什么要判断mActive是否为null呢？因为SerialExecutor想要实现的是模拟单一线程池的效果，即多个任务按照顺序一个一个执行。（因为sDefaultExecutor是一个静态变量，多个AsyncTask对象共用一个任务队列，所以这里的多个任务指的是：多个AsyncTask对象执行execute方法，而不是一个AsyncTask对象调用execute多次，前面分析过了，一个AsyncTask对象只能调用execute一次。）

    这样，第一个任务执行的时候，mActive为null，这时会进入if分支执行scheduleNext方法，对mActive进行赋值，并将Runnable扔到线程池中执行；而之后的任务执行的时候，mActive不为null，就不会进入if分支。

    那么，之后的任务该怎么执行呢？这就是为什么要包装一层了。从代码8~12可以看到，每个任务执行完毕、或者出错，最终都会调用scheduleNext方法来执行之后的任务，直到队列中没有任务。
